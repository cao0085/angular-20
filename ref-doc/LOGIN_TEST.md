## 功能說明

### 已實作功能
1. ✅ 帳號密碼驗證（使用 `user.json`）
2. ✅ 登入成功後儲存使用者資訊到 localStorage
3. ✅ 登入成功後儲存 Token
4. ✅ 錯誤訊息顯示（帳號或密碼錯誤）
5. ✅ 表單驗證（帳號至少3字元，密碼至少6字元）
6. ✅ 角色權限管理（根據 user.json 的 role 欄位）

### 登入流程
1. 使用者輸入帳號密碼
2. 點擊「登入」按鈕
3. 系統從 `user.json` 驗證帳號密碼
4. 驗證成功：
   - 儲存使用者資訊和 Token
   - 導向主頁面 (`/`)
5. 驗證失敗：
   - 顯示錯誤訊息「帳號或密碼錯誤」

## 測試步驟

1. 啟動開發伺服器
2. 導航到登入頁面 (`/login`)
3. 嘗試使用錯誤的帳號密碼 → 應該顯示錯誤訊息
4. 使用正確的帳號密碼（如 `admin` / `admin123`）
5. 登入成功後應該導向主頁面
6. 檢查 localStorage 是否儲存了使用者資訊和 Token

## 注意事項

- 目前使用的是 mock 資料（`user.json`），實際上線時需要替換為真實的 API 呼叫
- Token 是從 `user.json` 讀取的 mock token，實際應該由後端 API 產生
- 密碼是明文儲存在 JSON 中，僅供開發測試使用
